Crowd Stock 프로젝트 설계 문서 v0.1
프로젝트 목표
기존 투자 앱처럼 종목을 추천하거나 주가를 예측하는 앱이 아니다.
공개된 시장 데이터를 분석하여 시장 참여도를 관찰할 수 있도록 돕는 앱을 만든다.
________________________________________
프로젝트 비전
우리는 시장을 더 잘 예측하려는 앱이 아니라, 시장을 더 잘 관찰할 수 있게 만드는 앱을 만든다.
________________________________________
프로젝트 헌장
1.우리는 예측하지 않는다.
________________________________________
2.모든 지표는 공개된 시장 데이터를 기반으로 한다.
________________________________________
3.Crowd Score는 참고 지표이다.
투자 판단은 사용자가 직접 한다.
________________________________________
4.점수보다 근거를 먼저 보여준다.
사용자는 왜 점수가 높아졌는지, 왜 낮아졌는지를 이해할 수 있어야 한다.
________________________________________
5.단순함을 유지한다.
복잡한 기능보다 이해하기 쉬운 관찰 도구를 만든다.
________________________________________
앱 소개 문구(초안)
예측보다 관찰.
Crowd Stock은 공개된 시장 데이터를 기반으로 현재 시장 참여도를 추정하여 보여주는 투자 보조 앱입니다.
Crowd Score는 참고 지표이며, 투자 판단의 근거가 아닌 관찰을 위한 정보입니다.
________________________________________
시장 참여도 정의
시장 참여도란 공개된 시장 데이터를 기반으로, 기계적·제도적 거래의 영향을 가능한 범위에서 줄이고, 실제 투자자의 시장 참여 정도를 추정하여 나타낸 관찰 지표이다.
※ 여기서 “실제 투자자”는 개인만을 의미하지 않으며, 사람의 투자 판단에 의해 이루어진 거래를 최대한 반영하려는 개념이다.
________________________________________
현재 Crowd Engine 설계 방향
공식을 먼저 만들지 않는다.
다음 순서로 개발한다.
1.	관찰 요소 정의
2.	필요한 KIS 데이터 선정
3.	각 요소별 점수 계산
4.	Crowd Score 공식 작성
________________________________________
현재까지 정리된 관찰 요소(초안)
① 참여 규모 (Participation)
•	거래량
•	거래대금
________________________________________
② 참여 증가 (Acceleration)
•	거래량 증가율
•	거래대금 증가율
________________________________________
③ 참여 확산 (Spread)
•	회원사 분산도
•	투자자 구성
________________________________________
④ 참여 관심도 (가칭)
기계적 거래를 제외했을 때 실제 시장 참여를 얼마나 반영하는가.
후보 데이터
•	프로그램매매
•	회원사 매매
•	ETF 영향
※ 명칭은 추후 변경 가능
________________________________________
⑤ 참여 지속성 (Persistence)
관심이 일시적인지, 지속되는지를 관찰한다.
________________________________________
중요한 개발 철학
우리는 API 하나로 판단하지 않는다.
여러 API를 조합하여 관찰한다.
예시 1
•	프로그램매매 ↑
•	회원사 집중 ↑
•	거래량 급증
→ 기계적 거래 영향 가능성
예시 2
•	프로그램매매 ↓
•	회원사 분산 ↑
•	거래량 증가
→ 실제 시장 참여 증가 가능성
즉,
데이터 → 해석이 아니라,
데이터들의 조합 → 관찰을 만든다.
